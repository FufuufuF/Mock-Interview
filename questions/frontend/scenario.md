# 场景题与手写题

## 手写题

### Q1: 手写 Promise（包括 then/catch/all/race）
**难度**: 困难
**考察点**: Promise 原理、异步编程、微任务队列、链式调用
**追问方向**:
- Promise.allSettled 和 Promise.all 有什么区别？如何实现 allSettled？
- then 的链式调用中，如何处理返回值也是一个 Promise 的情况？
- 如何实现 Promise 的并发控制（例如最多同时执行 N 个请求）？

### Q2: 手写防抖和节流
**难度**: 基础
**考察点**: 闭包、定时器、this 指向、高阶函数
**追问方向**:
- 防抖的 immediate 模式（立即执行版本）如何实现？
- 节流的时间戳版本和定时器版本有什么区别？能否合并两种实现？
- 在 React 中使用防抖/节流需要注意什么？如何结合 useCallback 和 useRef 正确使用？

### Q3: 手写深拷贝
**难度**: 中等
**考察点**: 数据类型判断、递归、循环引用处理、WeakMap
**追问方向**:
- 如何处理 Date、RegExp、Map、Set 等特殊对象的拷贝？
- 循环引用的检测和处理思路是什么？为什么用 WeakMap 而不是 Map？
- structuredClone、JSON.parse(JSON.stringify()) 和手写深拷贝各有什么优缺点？

### Q4: 手写 EventEmitter / 发布订阅模式
**难度**: 中等
**考察点**: 设计模式、观察者模式与发布订阅的区别、事件机制
**追问方向**:
- once（只触发一次）如何实现？
- 观察者模式和发布订阅模式的本质区别是什么？
- 如何实现带命名空间的事件系统（例如 event.bindTo）？实际项目中如何防止内存泄漏？

### Q5: 手写 bind / call / apply
**难度**: 中等
**考察点**: this 绑定规则、原型链、函数柯里化、new 操作符
**追问方向**:
- 手写的 bind 返回的函数如果被 new 调用，this 应该指向谁？如何处理？
- call 和 apply 的实现中，如何处理传入 null 或 undefined 作为 context 的情况？
- ES6 的箭头函数能否被 bind？为什么？

### Q6: 手写 instanceof
**难度**: 基础
**考察点**: 原型链、__proto__、prototype、类型判断
**追问方向**:
- instanceof 能否正确判断基本类型？如何让它支持基本类型（Symbol.hasInstance）？
- 在多 iframe 环境下 instanceof 会出现什么问题？如何解决？
- 如何实现一个更可靠的类型判断函数？Object.prototype.toString.call 的原理是什么？

### Q7: 手写数组扁平化
**难度**: 基础
**考察点**: 递归、迭代、reduce、数组方法
**追问方向**:
- 如何实现指定深度的扁平化（类似 Array.prototype.flat(depth)）？
- 除了递归，还有哪些实现方式？（栈迭代、toString、Generator）
- 如果数组非常大且嵌套很深，递归方案有什么问题？如何优化？

### Q8: 手写柯里化函数
**难度**: 中等
**考察点**: 闭包、递归、函数式编程、参数收集
**追问方向**:
- 柯里化和偏函数（partial application）有什么区别？
- 如何实现一个支持占位符的柯里化函数？
- 柯里化在实际业务中有哪些应用场景？在函数组合（compose/pipe）中如何配合使用？

### Q9: 实现一个简单的响应式系统
**难度**: 困难
**考察点**: Proxy/Object.defineProperty、依赖收集、发布订阅、Vue 响应式原理
**追问方向**:
- Object.defineProperty 和 Proxy 实现响应式各有什么优缺点？Vue 2 到 Vue 3 为何切换？
- 依赖收集的过程是怎样的？effect、track、trigger 分别做了什么？
- 如何处理嵌套对象的响应式？如何实现 computed 的懒求值和缓存？

### Q10: 实现 LRU 缓存
**难度**: 中等
**考察点**: Map 有序性、双向链表、哈希表、缓存淘汰策略
**追问方向**:
- 如果不用 Map，如何用双向链表 + 哈希表实现 O(1) 的 get 和 put？
- LRU 和 LFU 的区别是什么？各自适用什么场景？
- 在前端实际项目中，LRU 缓存有哪些应用场景？（接口缓存、组件缓存、keep-alive）

## 场景设计题

### Q11: 大文件上传方案设计
**难度**: 困难
**考察点**: 文件分片、断点续传、并发控制、Web Worker、Hash 计算
**追问方向**:
- 如何计算文件的唯一标识（Hash）？大文件 Hash 计算如何避免阻塞主线程？
- 断点续传的具体实现流程是什么？前后端分别需要做哪些工作？
- 分片上传的并发数如何控制？上传失败的分片如何重试？秒传功能如何实现？

### Q12: 虚拟列表 / 长列表渲染优化
**难度**: 困难
**考察点**: DOM 性能优化、滚动事件处理、可视区域计算、IntersectionObserver
**追问方向**:
- 定高和不定高列表项的虚拟滚动实现有什么区别？不定高如何处理？
- 如何处理快速滚动时的白屏问题？缓冲区（buffer）的作用是什么？
- 虚拟列表中如何支持搜索、跳转到指定位置等功能？

### Q13: 前端权限控制方案
**难度**: 中等
**考察点**: RBAC 模型、路由权限、按钮权限、菜单权限、数据权限
**追问方向**:
- 前端路由权限有哪几种实现方式？动态路由和静态路由过滤各有什么优缺点？
- 按钮级别的权限控制如何实现？自定义指令（v-permission）和组件封装哪种更好？
- 前端权限控制能否保证安全？为什么后端也必须做权限校验？

### Q14: 前端错误监控方案
**难度**: 中等
**考察点**: 错误捕获、SourceMap、性能监控、上报策略、用户行为追踪
**追问方向**:
- JS 运行时错误、Promise 未捕获异常、资源加载错误分别如何捕获？
- 线上环境如何利用 SourceMap 还原错误堆栈？SourceMap 的安全性如何保证？
- 错误上报的策略有哪些？如何避免大量重复错误导致的上报风暴？

### Q15: 前端性能优化全链路方案
**难度**: 困难
**考察点**: 加载性能、运行时性能、Core Web Vitals、构建优化、缓存策略
**追问方向**:
- 首屏加载优化有哪些手段？如何量化首屏时间（FCP、LCP）？
- 长任务（Long Task）如何检测和优化？如何利用 requestIdleCallback 和时间切片？
- 从输入 URL 到页面渲染完成，每个阶段各有哪些优化手段？

### Q16: 前端微前端架构设计
**难度**: 困难
**考察点**: 应用隔离、样式隔离、JS 沙箱、通信机制、路由分发
**追问方向**:
- qiankun、micro-app、Module Federation 各有什么优缺点？如何选型？
- JS 沙箱的实现原理是什么？快照沙箱和 Proxy 沙箱有什么区别？
- 微前端之间如何实现高效的状态共享和通信？

### Q17: 实时协同编辑方案设计
**难度**: 困难
**考察点**: OT 算法、CRDT、WebSocket、冲突解决、操作变换
**追问方向**:
- OT（Operational Transformation）和 CRDT（Conflict-free Replicated Data Type）各自的原理和优缺点是什么？
- 多人同时编辑同一段文字时，如何保证最终一致性？
- 离线编辑后重新上线的冲突如何处理？光标和选区的同步如何实现？
