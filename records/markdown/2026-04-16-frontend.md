## 题目与回答

### Q1: var、let、const 三者的区别
- **评分**: A
- **回答摘要**: 从作用域（函数级 vs 块级）、可修改性、全局变量挂载 window、变量提升与暂时性死区四个维度详细说明，提到 let/const 编译角度也存在提升但指向"魔法值"。
- **追问**: const 声明的对象属性能否修改？如何实现不可变对象？Object.freeze 是深冻结还是浅冻结？
- **面试官反馈**: 回答准确且有深度，覆盖核心知识点，对 TDZ 的理解超出基础水平。追问部分推理正确。

### Q2: 闭包的概念和应用场景
- **评分**: B
- **回答摘要**: 描述了函数 A 返回函数 B 且 B 引用了 A 的变量时变量不会被销毁的行为，提到回调函数和模块机制依赖闭包。
- **追问**: 闭包会不会导致内存泄漏？
- **面试官反馈**: 核心概念正确但定义偏窄，闭包不限于"函数返回函数"。应用场景需补充：防抖节流、柯里化、私有变量封装等。内存泄漏的实际场景举例不够具体。

### Q3: 事件循环机制（宏任务与微任务）
- **评分**: A
- **回答摘要**: 准确描述宏任务→微任务→渲染的循环流程，微任务队列完全清空才进入下一环节，宏任务队列有多个而微任务队列只有一个。代码输出题回答正确（3 2 1）。
- **追问**: Promise.then 回调中再创建 Promise.then，新微任务在 setTimeout 之前还是之后？
- **面试官反馈**: 回答准确全面，对执行顺序的理解清晰。

### Q4: 重排与重绘
- **评分**: C
- **回答摘要**: 修改几何尺寸触发重排，修改颜色等触发重绘，重排性能消耗大于重绘。不了解读取 offsetWidth 等属性会触发重排。
- **追问**: 为什么读取 offsetWidth/clientHeight 会触发重排？
- **面试官反馈**: 基础概念正确但不够深入。需补充强制同步布局（Forced Synchronous Layout）的概念。

### Q5: BFC
- **评分**: D
- **回答摘要**: 表示 CSS 基本不了解，跳过。
- **追问**: 无
- **面试官反馈**: BFC 是高频考点，需系统学习触发条件和解决的问题（margin 折叠、清除浮动、高度塌陷）。

### Q6: 强缓存与协商缓存
- **评分**: B
- **回答摘要**: 描述了 Expires 到 Cache-Control 的演进，协商缓存的两对请求/响应头和 304 状态码。不了解 no-cache 和 no-store 的区别。
- **追问**: Cache-Control 中 no-cache 和 no-store 的区别？
- **面试官反馈**: 整体框架正确，部分术语不准确（响应头名称、"预检请求"应为"条件请求"）。

### Q7: 跨域与同源策略
- **评分**: B
- **回答摘要**: 正确定义同源（协议+域名+端口），说明了跨域时浏览器拦截响应，同源策略限制跨域请求、Cookie 读取、iframe 操作。
- **追问**: CORS 简单请求和预检请求的触发条件？
- **面试官反馈**: 核心概念正确。简单请求的触发条件回答不够精确，不是按"是否破坏性"判断，而是看方法、Content-Type 和自定义请求头。

### Q8: HTTPS TLS 握手
- **评分**: A
- **回答摘要**: 完整描述客户端发送加密算法+随机数A → 服务端返回算法+随机数B+证书 → 客户端验证证书加密预主密钥 → 生成对称密钥的全过程。
- **追问**: 无
- **面试官反馈**: TLS 握手流程理解清晰准确，混合加密核心逻辑掌握良好。

### Q9: useState 工作原理与 Hooks 调用规则
- **评分**: A
- **回答摘要**: useState 创建 hook 节点，setState 将状态推入队列渲染时统一更新，Hooks 通过链表组织因此不能在条件/循环中调用。
- **追问**: setState 是同步还是异步？React 18 之后有什么变化？
- **面试官反馈**: 回答准确，对底层链表结构和批量更新机制的演进理解深入。

### Q10: useMemo、useCallback 与 React.memo
- **评分**: A
- **回答摘要**: useCallback 稳定函数引用，useMemo 稳定计算值引用，React.memo 让组件在 props 引用不变时跳过重渲染。提到不应滥用，浅比较本身有开销。
- **追问**: 是否应该给每个函数都包裹 useCallback？
- **面试官反馈**: 理解全面且准确，能从性能权衡角度思考。
