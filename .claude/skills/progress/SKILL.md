# 面试进度统计

你是一位数据分析师，需要分析候选人的模拟面试历史数据，生成进度报告。

## 操作步骤

### 1. 收集数据

- 使用 Glob 列出 `records/json/*.json` 下所有文件
- 逐个读取所有 JSON 记录文件
- 如果没有任何记录，提示用户先使用 `/interview` 进行面试

### 2. 数据统计

需要统计以下维度：

**总体数据**:
- 总面试次数
- 覆盖的方向数
- 总题目数
- 各等级题目占比 (A/B/C/D)

**按方向统计**:
- 每个方向的面试次数
- 每个方向的平均得分趋势（按时间排序）
- 每个方向最近一次的综合评分

**知识点分析**:
- 从 tags 字段提取所有知识点标签
- 找出反复评分为 C/D 的知识点（出现 2 次及以上）
- 找出从低分进步到高分的知识点

### 3. 输出进度报告

```
# 模拟面试进度报告

**统计周期**: [最早日期] — [最晚日期]
**总面试次数**: N 次
**总题目数**: N 道

## 总体评分分布

| 等级 | 数量 | 占比 |
|------|------|------|
| A    | N    | XX%  |
| B    | N    | XX%  |
| C    | N    | XX%  |
| D    | N    | XX%  |

## 各方向进展

### [方向名称]
- 面试次数: N
- 得分趋势: D → C → B（按时间顺序）
- 最近评分: X
- 状态: [进步中 / 稳定 / 需加强]

（每个方向...）

## 反复出错的知识点（重点关注）

以下知识点在多次面试中持续得分较低：

1. **[知识点]** — 出现 N 次，平均评分 X，最近评分 X
2. **[知识点]** — 出现 N 次，平均评分 X，最近评分 X

## 明显进步的知识点

1. **[知识点]** — 从 D 提升到 B
2. **[知识点]** — 从 C 提升到 A

## 学习优先级建议

根据数据分析，建议按以下优先级进行复习：

1. **[方向/知识点]** — 原因
2. **[方向/知识点]** — 原因

## 下一步计划

建议接下来进行以下方向的面试：
- /interview [方向] — 原因
```

### 注意事项
- 如果记录太少（< 3 次），说明数据量不足以看出趋势，给出初步分析即可
- 得分转换规则：A=4, B=3, C=2, D=1，用于计算平均分和趋势
- 趋势判断：连续 2 次提升为"进步中"，连续 2 次下降为"需加强"，否则"稳定"
